# 竞品调研网站 · 产品需求

> 本文档定义产品功能需求，是开发的依据。每次开发前请对照本文档。

## 一、核心价值

打通"知道→做到"的信息差链路：
- 知道：自动收集、整合、提取现有信息
- 做到：输出行动方案

## 二、用户场景

### 场景 1：个人了解行业
> "我想知道 AI 行业最近半年发生了什么"
- 输入"AI 行业趋势"
- 系统搜索近半年的信息
- 生成时间线：GPT-5 发布、欧盟 AI 法案生效、Anthropic 融资...
- 输出报告：关键趋势 + 行动建议

### 场景 2：企业竞品调研
> "我想了解竞品 Notion 过去一年做了什么"
- 输入"Notion 产品动态"
- 系统搜索并整理
- 生成报告：产品更新时间线 + 功能变化 + 市场策略

### 场景 3：持续追踪
> "我想每周自动追踪深圳经济政策变化"
- 订阅"深圳经济政策"主题，设为每周
- 系统每周自动搜索新信息
- 新事件自动入库
- 可随时查看更新报告

### 场景 4：对话式深入分析
> "GPT-5 发布对个人开发者意味着什么？"
- 基于已收集的事件数据
- AI 对话回答，提供上下文相关的建议

## 三、功能需求

### 3.1 用户系统
| 功能 | 说明 | 优先级 |
|---|---|---|
| 注册 | 邮箱+密码注册 | 必须 |
| 登录 | 邮箱+密码登录，获取 JWT 令牌 | 必须 |
| 获取当前用户 | 验证令牌，返回用户信息 | 必须 |

### 3.2 主题管理 (Topic)
| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建主题 | 输入关键词创建一个追踪主题 | 必须 |
| 查看主题列表 | 列出当前用户的所有主题 | 必须 |
| 查看主题详情 | 显示主题的事件时间线 | 必须 |
| 更新主题 | 修改名称/描述 | 可选 |
| 删除主题 | 删除主题及其所有数据 | 必须 |

### 3.3 信息采集与研究 (Research)
| 功能 | 说明 | 优先级 |
|---|---|---|
| 触发搜索 | 对主题执行一次搜索+AI分析 | 必须 |
| 时间范围筛选 | 支持按天/周/月/年搜索 | 必须 |
| 自动去重 | 相同标题的事件不重复入库 | 必须 |
| 事件分类 | 自动标记类别：产品/政策/财务/技术等 | 必须 |

### 3.4 报告生成 (Report)
| 功能 | 说明 | 优先级 |
|---|---|---|
| 生成报告 | 基于主题事件，AI 生成总结报告 | 必须 |
| 查看报告列表 | 列出主题下的所有报告 | 必须 |
| 查看单个报告 | 显示报告全文 | 必须 |
| 报告内容 | 标题 + 正文(Markdown) + 行动建议 + 关键发现 | 必须 |

### 3.5 订阅系统 (Subscription)
| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建订阅 | 对主题设置定时追踪 | 必须 |
| 查看订阅列表 | 列出所有订阅及状态 | 必须 |
| 暂停订阅 | 临时停止追踪 | 必须 |
| 恢复订阅 | 恢复追踪 | 必须 |
| 删除订阅 | 彻底删除 | 必须 |
| 自动执行 | 每天/每周定时自动搜索新信息 | 必须 |

### 3.6 对话系统 (Chat)
| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建对话 | 开启一个新的对话会话 | 必须 |
| 查看对话列表 | 列出历史对话 | 必须 |
| 发送消息 | 用户提问，AI 回答 | 必须 |
| 查看历史消息 | 显示某个对话的所有消息 | 必须 |
| 上下文记忆 | AI 基于对话历史回答 | 必须 |

## 四、数据模型

### 主题 (Topic)
- 名称、描述、所属用户

### 事件 (Event)
- 日期、标题、摘要、类别、来源
- 归属某个主题

### 报告 (Report)
- 类型(摘要/时间线/行动方案)、标题、正文、结构化数据
- 归属某个主题

### 订阅 (Subscription)
- 频率(每天/每周/每月)、是否活跃、上次运行时间
- 关联主题和用户

### 对话 (ChatSession + ChatMessage)
- 会话：标题、关联主题
- 消息：角色(用户/AI)、内容、时间

## 五、非功能需求

| 项目 | 要求 |
|---|---|
| 响应速度 | 搜索+AI分析在 30 秒内完成 |
| 可用性 | 页面加载 < 3 秒 |
| 数据安全 | 密码加密存储，JWT 令牌认证 |
| 代码质量 | 有测试覆盖，遵循 TDD |

## 六、设计要求

- 风格：水墨文学风
- 配色：宣纸白(#f5f0e8) + 墨色(#1a1a2e) + 青瓷绿(#5a7d6e)
- 字体：思源宋体 + Cormorant Garamond
- 交互：方角边框，印章式 Logo，竖线时间轴
