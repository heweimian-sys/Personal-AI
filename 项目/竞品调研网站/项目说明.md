# 竞品调研网站 · 项目说明

> 本文档是项目的唯一事实来源，每次开发前请先阅读本文档。

## 一、项目是什么

一个面向中文用户的信息整理平台。

你输入任意关键词（比如"AI 行业趋势""美食演变""深圳经济政策""某竞品动态"），平台会：

1. 自动搜索互联网上的相关信息（使用 Firecrawl 自部署，完全免费无限制）
2. 用 AI 把信息整理成时间线（按日期排列的事件）
3. 分类标记（产品、政策、行业、融资、技术等）
4. 建立主题之间的双向链接（类似 Obsidian，主题可互相关联）
5. 生成结构化报告和行动建议

**一句话总结**：输入关键词，了解过去和现在发生了什么，发现趋势，获得行动建议，主题之间互相关联。

**不只是竞品调研**：这个平台可以做竞品分析，也可以做趋势梳理、选题管理、知识演变追踪。搜索"美食"能看到从古至今的演变，搜索"AI"可以做技术梳理。

## 二、为谁做

- **面向中文用户**：所有界面、文档、AI 交互均使用中文
- **无网络限制**：使用国内可直连的服务商，不需要翻墙
- **个人**：了解某个领域/行业的历史和现状，推演趋势
- **企业**：一份自动生成的竞品调研报告

## 三、使用场景

| 场景 | 关键词示例 | 输出 |
|---|---|---|
| 竞品分析 | "Notion 产品动态" | 产品更新时间线 + 功能变化 + 市场策略 |
| 趋势梳理 | "AI 行业趋势" | 关键事件时间线 + 技术趋势报告 |
| 选题管理 | "短视频赛道" | 行业事件 + 热点分析 + 选题建议 |
| 知识演变 | "美食的演变" | 从古至今的发展脉络 + 对人的影响 |
| 政策追踪 | "深圳经济政策" | 政策时间线 + 影响分析 |
| 趋势推演 | "新能源车市" | 历史事件链 + 关联主题 + 未来推演 |

## 四、核心功能

### 双向链接（类似 Obsidian）

这是平台的**核心差异化功能**，不是附属功能：

- 主题之间可以互相关联
- 在事件内容中用 `[[主题名]]` 标记关联
- 被关联的主题自动显示反向链接
- 浏览主题时可以快速跳转到关联主题
- 发现肉眼不可见的趋势关联

## 五、技术栈

| 层面 | 技术 | 为什么选它 |
|---|---|---|
| 后端语言 | Python 3.12 | 生态好，AI 库丰富 |
| 后端框架 | FastAPI | 速度快，自动生成 API 文档 |
| 数据库 | PostgreSQL 16 | 稳定可靠，适合存储结构化数据 |
| AI 模型 | DeepSeek | 国内可直连，性价比高，支持 JSON 输出 |
| 搜索引擎 | Firecrawl（自部署） | 完全免费，无限制，爬取+结构化 |
| 前端框架 | Next.js 15 (React) | 主流，生态成熟 |
| 认证 | 访问密码 | MVP 阶段简化，不做多用户 |
| 部署 | Docker（后续） | MVP 先本地跑通 |

## 六、设计风格
