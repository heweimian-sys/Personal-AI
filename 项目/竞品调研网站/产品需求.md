# 知信 · 认知加速器 — 产品需求

> 本文档定义产品功能需求，是开发的依据。每次开发前请对照本文档。
> v5 版本

## 一、核心价值

**打通"知道→做到"的信息差链路**：

- 知道：自动搜索、整合、提取信息
- 理解：分析事件间的因果脉络，发现肉眼不可见的趋势
- 做到：输出分角色的行动建议

**核心痛点**：信息有，但脑子进不去。

## 二、信息摄入的心理模型

```
好奇心 → 故事线 → 因果链 → 全局感 → 行动欲
  ↓        ↓        ↓        ↓        ↓
引导语   分章节   连接线   留白消化   总结判断
```

每一步都是一个心理转换，UI 服务于这个节奏。

## 三、用户场景

### 场景 1：个人求知
> "我想了解 AI 行业发生了什么"
- 输入"AI行业"
- 开头引导：AI 生成的 2-3 句摘要
- 章节式阅读：第一章·模型之争 → 第二章·应用爆发
- 因果连接：事件间用线+箭头连接，标注"直接导致""引发"
- 侧边注释：补充数据来源和关联信息
- 结尾洞察：三个关键判断 + 分角色行动建议

### 场景 2：企业调研
> "我想了解新能源政策走向"
- 输入"新能源政策"
- 政策时间线 + 因果脉络 + 竞品格局
- 行动建议：投资视角、创业视角、求职视角

### 场景 3：自媒体选题
> "我想做美食文化的选题"
- 输入"美食文化"
- 历史变迁脉络 + 影响因素分析
- 选题角度建议

### 场景 4：趋势推演
> "半导体产业链会怎么走"
- 输入"半导体产业链"
- 事件因果链 + 关联主题（包含关系、技术依赖）
- 未来推演判断

## 四、Phase 1 MVP 功能范围

### 4.1 搜索入口

| 功能 | 说明 | 优先级 |
|---|---|---|
| 关键词输入 | 搜索框 + 示例标签 | 必须 |
| 关键词建议 | 试试：AI行业 \| 新能源 \| 半导体 \| 美食文化 | 必须 |

### 4.2 信息采集与研究

| 功能 | 说明 | 优先级 |
|---|---|---|
| 触发搜索 | 对关键词执行搜索+AI分析 | 必须 |
| Tavily 搜索 | 免费 1000 次/月，专为 AI 设计（优先） | 必须 |
| AI 事件提取 | DeepSeek 提取结构化事件 | 必须 |
| AI 关系分析 | 分析事件间因果关系 | 必须 |
| AI 洞察生成 | 生成趋势判断和行动建议 | 必须 |

### 4.3 阅读式信息流展示

| 功能 | 说明 | 优先级 |
|---|---|---|
| 开头引导摘要 | 2-3 句 AI 摘要卡片 | 必须 |
| 章节式叙事 | 2-4 章节，每章 2-4 事件 | 必须 |
| 因果连接线 | 事件卡片间细线+箭头，标注关系类型 | 必须 |
| 侧边注释 | 右侧 260px 放关联信息、数据来源 | 必须 |
| 留白节奏 | 章节间 120px+ 间距 | 必须 |
| 结尾洞察 | AI 判断 + 分角色行动建议 | 必须 |
| 操作区 | 导出报告、追踪此主题 | 必须 |

### 4.4 响应式设计

| 屏幕 | 布局 |
|---|---|
| ≥1024px | 正文 720px + 右侧侧边注 |
| 768-1024px | 正文 100%，侧边注改为行内注释 |
| <768px | 单列，侧边注消失，连接线简化 |

## 六、数据模型

```python
class Event:
    """信息事件节点"""
    id: str
    title: str                    # "GPT-5 发布，推理速度提升3倍"
    summary: str                  # AI 生成摘要
    date: datetime
    sources: list[Source]         # 引用来源
    confidence: float             # 确信度 0-1
    key_quote: str | None         # 关键引述

class Topic:
    """主题/实体"""
    id: str
    name: str                    # "大模型军备竞赛" / "OpenAI"
    type: str                    # 主题 / 公司 / 人物 / 技术 / 政策
    description: str

class Relation:
    """关联边"""
    id: str
    from_id: str
    to_id: str
    type: str                    # causal / competitive / contains / dependency / chain
    description: str             # "Claude 4 的竞争压力迫使 OpenAI 提前发布 GPT-5"
    confidence: float

class Insight:
    """AI 洞察"""
    id: str
    title: str                   # "模型层战争结束，应用层刚刚开始"
    body: str                    # 详细分析
    judgments: list[str]         # 关键判断
    suggestions: dict[str, list[str]]  # 按角色分组的行动建议
    related_event_ids: list[str]
    related_relation_ids: list[str]

class Report:
    """一次搜索生成的完整报告"""
    id: str
    query: str                   # 用户输入的关键词
    summary: str                 # 开头引导摘要
    chapters: list[Chapter]      # 章节
    events: list[Event]          # 所有事件
    relations: list[Relation]    # 所有关系
    insights: list[Insight]      # 所有洞察
    generated_at: datetime
```

## 八、AI 处理流程

```
用户输入关键词
    │
    ▼
Tavily 搜索 → 获取原始信息列表（降级链：Tavily → Firecrawl → 兜底数据）
    │
    ▼
DeepSeek #1 → 提取结构化事件
  (标题、摘要、日期、来源、关键引述)
    │
    ▼
DeepSeek #2 → 分析事件关联
  (因果关系、竞争关系、包含关系、连锁反应)
    │
    ▼
DeepSeek #3 → 生成洞察和行动建议
  (趋势判断、分角色建议)
    │
    ▼
组装 Report JSON → 返回前端
```
