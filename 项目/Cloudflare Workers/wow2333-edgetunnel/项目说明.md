# wow2333 Edgetunnel Worker

> Cloudflare Worker: `bold-poetry-bb94`  
> Admin: `https://wow2333.us.ci/admin`  
> Upstream: `https://github.com/cmliu/edgetunnel`

## 目标

把当前 Cloudflare Worker 的代码纳入 GitHub 仓库管理，并实现：

- `main` 分支更新后自动部署到 Cloudflare Worker。
- 每天自动检查上游 `cmliu/edgetunnel` 的 `_worker.js`，如果有更新，就提交到本仓库并部署。
- 保留 Cloudflare Dashboard 中已有的环境变量，例如 `ADMIN`、`KEY`、`UUID` 等敏感配置。

## 文件说明

| 文件 | 用途 |
|---|---|
| `_worker.js` | Worker 主代码，来自 `cmliu/edgetunnel` |
| `wrangler.toml` | Cloudflare Worker 部署配置 |
| `package.json` | 本地部署/检查命令 |
| `scripts/check-config.mjs` | 部署前检查 KV 配置是否已经填写 |

## 首次配置

### 1. 配置 GitHub Secrets

在 `heweimian-sys/Personal-AI` 仓库里进入：

`Settings -> Secrets and variables -> Actions -> New repository secret`

新增：

| Secret | 说明 |
|---|---|
| `CLOUDFLARE_API_TOKEN` | Cloudflare API Token |

Cloudflare Account ID 已经写入 `wrangler.toml` 和 GitHub Actions：

```text
bce51bb1789dc3f9996d751ddac0591e
```

API Token 建议只给 Workers 编辑权限，并限定到你的 Cloudflare 账号。

建议 Token 权限：

| 权限 | 级别 |
|---|---|
| Account -> Workers Scripts | Edit |
| Account -> Workers KV Storage | Read |
| Zone -> Workers Routes | Edit，仅当这个 Worker 需要管理路由时添加 |

如果 Cloudflare 提示权限不足，再按提示补权限，不要使用全局 API Key。

### 2. 确认 Cloudflare 运行时变量

`wrangler.toml` 设置了 `keep_vars = true`，用于保留 Dashboard 里的环境变量。你当前 Worker 里已经有：

- `ADMIN`: 后台登录密码。
- `UUID`: 当前节点 UUID。

不要把这两个值写进仓库。

### 3. KV 绑定

新版 edgetunnel 的 `/admin` 后台需要 KV 才能正常显示。当前绑定：

| 项目 | 值 |
|---|---|
| Binding name | `KV` |
| Namespace | `wow2333-edgetunnel` |
| Namespace ID | `08945abd7dd54fdfb62a7484b7f0549a` |

不要修改 binding name，代码里固定读取 `env.KV`。

## 自动更新逻辑

`.github/workflows/deploy-wow2333-worker.yml` 会处理三种情况：

- push 到 `main` 且改动本目录时：部署到 Cloudflare。
- 手动运行 workflow：部署当前代码。
- 每天定时运行：检查上游 `_worker.js`，有变化就提交并部署。

## 本地命令

```bash
npm install
npm run check:config
npm run deploy
```

本地部署需要先登录 Wrangler 或设置 Cloudflare API Token。

## 安全提醒

不要把后台密码、订阅 token、UUID、Cloudflare API Token 写进仓库。截图或公开分享后台页面前，也建议遮住节点链接和订阅地址。

## 关于上游 README 的“删除部署”

上游项目 README 里提到“建议测试完成后 24 小时内删除本项目相关部署”。这里的“部署”指 Cloudflare Worker/Pages 上已经发布出来的服务，不是 GitHub 仓库，也不是 Actions 的部署记录。

删除 Worker/Pages 部署后，`https://wow2333.us.ci/admin` 会停止工作。当前项目是你自己的长期站点，不需要因为这句话自动删除部署；是否继续使用取决于你自己对当地法律法规、Cloudflare 服务条款和项目用途的判断。
