# 竞品调研网站 · 产品需求

> 本文档定义产品功能需求，是开发的依据。每次开发前请对照本文档。
> 本版为 v2 MVP 版本，聚焦搜索式核心链路，订阅与对话移入 Backlog。

## 一、核心价值

打通"知道→做到"的信息差链路：

- 知道：自动收集、整合、提取现有信息
- 做到：输出行动方案

## 二、用户场景

### 场景 1：知识演变追踪
> "我想知道美食从古至今是怎么演变的"
- 输入"美食的演变"
- 系统搜索相关信息
- 生成时间线：古代烹饪方式 → 工业革命食品加工 → 现代外卖文化
- 输出报告：演变脉络 + 对人的影响

### 场景 2：竞品调研
> "我想了解竞品 Notion 过去一年做了什么"
- 输入"Notion 产品动态"
- 系统搜索并整理
- 生成报告：产品更新时间线 + 功能变化 + 市场策略

### 场景 3：趋势梳理
> "我想了解 AI 行业最近半年的趋势"
- 输入"AI 行业趋势"
- 生成时间线：GPT-5 发布、DeepSeek 开源、欧盟 AI 法案...
- 输出报告：关键趋势 + 行动建议

### 场景 4：选题管理
> "我想做短视频赛道的选题"
- 输入"短视频赛道"
- 生成时间线：平台政策变化、热点事件、创作者动态
- 输出报告：热点分析 + 选题建议

## 三、MVP 功能范围

### 3.1 访问控制（简化版）

| 功能 | 说明 | 优先级 |
|---|---|---|
| 密码登录 | 输入访问密码获取令牌 | 必须 |
| 令牌校验 | 所有 API 需要有效令牌 | 必须 |

> MVP 不做多用户注册登录，所有数据不挂 user_id。后续要多用户再加字段迁移。

### 3.2 主题管理 (Topic)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建主题 | 输入关键词创建一个追踪主题 | 必须 |
| 查看主题列表 | 列出所有主题 | 必须 |
| 查看主题详情 | 显示主题的事件时间线 | 必须 |
| 删除主题 | 删除主题及其所有数据 | 必须 |

### 3.3 信息采集与研究 (Research)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 触发搜索 | 对主题执行一次搜索+AI分析 | 必须 |
| 时间范围筛选 | 支持不限/一天/一周/一月/一年 | 必须 |
| 自动去重 | 相同标题的事件不重复入库 | 必须 |
| 事件分类 | 自动标记类别：产品/政策/行业/融资/其他 | 必须 |

### 3.4 报告生成 (Report)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 生成报告 | 基于主题事件，AI 生成总结报告 | 必须 |
| 查看报告列表 | 列出主题下的所有报告 | 必须 |
| 查看单个报告 | 显示报告全文 | 必须 |
| 报告内容 | 总览 + 关键趋势 + 行动建议 + Markdown 正文 | 必须 |

## 四、数据模型

### 主题 (Topic)
- keyword（搜索关键词）、display_name（显示名称）
- 预留 links 关系字段（为双向链接做扩展准备）

### 事件 (Event)
- occurred_at（发生日期）、title、summary、category、source
- 归属某个主题

### 报告 (Report)
- report_type（报告类型）、content_md（Markdown正文）、structured（结构化数据）
- 归属某个主题

> MVP 不包含 Subscription、ChatSession、ChatMessage 模型，这些在 Backlog 中。

## 五、技术约束

| 项目 | 要求 |
|---|---|
| 网络限制 | 必须使用国内可直连服务商（博查搜索 + DeepSeek AI） |
| 响应速度 | 搜索+AI分析在 30 秒内完成 |
| 可用性 | 页面加载 < 3 秒 |
| 代码质量 | 核心链路有测试覆盖（搜索服务/AI服务/研究管道） |
| 语言 | 所有界面、文档、AI 交互均使用中文 |

## 六、设计要求

- 风格：水墨文学风
- 配色：宣纸白(#f5f0e8) + 墨色(#1a1a2e) + 青瓷绿(#5a7d6e)
- 字体：思源宋体 + Cormorant Garamond
- 交互：方角边框，印章式 Logo，竖线时间轴

## 七、Backlog（验证通过后再启动）

以下功能本轮不做，先记录设计要点：

- **订阅式追踪**：APScheduler 定时任务 + Subscription 模型，给已有 topic 加"定时重复触发"开关
- **对话式问答**：ChatSession + ChatMessage 模型，基于已积累的 events/reports 做 RAG 问答
- **完整多用户认证**：Topic/Event/Report 模型加 user_id 外键，从访问密码升级为注册登录
- **双向链接**：`[[主题名]]` 语法解析 + TopicLink 表 + AI 智能推荐关联 + 知识图谱可视化
- **Docker Compose 部署**：backend + frontend + db 一起打包
- **可视化看板**：Recharts 图表（事件密度趋势、分类占比等）
- **内容合规**：敏感词过滤、来源标注/版权、ICP 备案
