## 2026-07-08 Task 2 — 数据模型

### 今日完成
- 创建 5 个数据模型 + 6 个 Pydantic Schema，搭建完整数据骨架
- 模型层：
  - `models/base.py` — Base 基类 + TimestampMixin（created_at, updated_at）
  - `models/topic.py` — 主题/实体（名称、类型：主题/公司/人物/技术/政策、描述）
  - `models/event.py` — 信息事件（标题、摘要、日期、来源JSON、确信度、关键引述）
  - `models/relation.py` — 关联边（5 种类型：causal/competitive/contains/dependency/chain，双 FK）
  - `models/insight.py` — AI 洞察（判断列表、分角色建议字典、关联事件/关系 ID）
  - `models/report.py` — 完整报告（查询词、摘要、章节 JSON、级联删除子记录）
  - `models/__init__.py` — 统一导入注册
- Schema 层：
  - `schemas/` 下 6 个文件，每个模型对应 Create + Response Schema
  - 额外定义 Source（信息来源）和 Chapter（章节结构）
- 架构调整：
  - `db.py` 中 Base 移至 `models/base.py`，db.py 改为导入
  - `import app.models` 确保所有模型注册到 Base.metadata
- 测试：10 个模型测试（CRUD + 5 种关系类型 + 级联删除）
- 测试结果：13 passed in 1.91s（含 Task 1 的 3 个健康检查测试）

### 数据模型关系图
```
Report (顶层)
  ├── Event[] (事件列表)
  ├── Relation[] (关系列表，双 FK 指向 Event)
  └── Insight[] (洞察列表)

Topic (独立实体，Phase 2+ 用于跨报告关联)
```
