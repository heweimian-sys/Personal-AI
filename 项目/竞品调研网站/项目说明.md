# 知信 · 认知加速器 — 项目说明

> 本文档是项目的唯一事实来源，每次开发前请先阅读本文档。
> v5 版本：从"信息聚合器"进化为"认知加速器"

## 一、项目是什么

**知信**——一个让信息像好文章一样自然流入大脑的阅读式信息流平台。

输入一个关键词，系统不只是返回链接列表，而是生成一篇结构化的深度分析：

1. 搜索互联网信息（Firecrawl 自部署，完全免费）
2. AI 提取结构化事件（DeepSeek，国内直连）
3. 分析事件间的因果关系（不只排列时间线，还分析"为什么会这样"）
4. 以章节式阅读体验展示（像读一篇深度报道）
5. 生成洞察判断和分角色行动建议

**一句话总结**：输入关键词，看到信息的因果脉络，获得可行动的判断。

## 二、解决什么痛点

> 信息有，但脑子进不去。

| 痛点 | 知信怎么解决 |
|---|---|
| 找不到信息 | Firecrawl 搜索，免费无限制 |
| 整合信息难 | AI 自动提取结构化事件，按叙事逻辑分章 |
| 梳理信息难 | 事件间因果连接线，侧边注释补充上下文 |
| 信息进不去脑子 | 阅读式信息流：引导语→章节→因果链→留白→洞察→行动 |

## 三、与普通搜索的区别

| | 搜索引擎 | 知信 |
|---|---|---|
| 给你什么 | 链接列表 | 结构化时间线 |
| 信息关系 | 无 | 因果关系/竞争关系/包含关系 |
| 阅读方式 | 碎片跳转 | 章节式深度阅读 |
| 输出 | 你自己判断 | AI 提炼洞察 + 行动建议 |

## 四、使用场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 个人求知 | "AI行业" | AI 发展脉络 + 趋势判断 |
| 企业调研 | "新能源政策" | 政策演变 + 竞品格局 + 机会判断 |
| 自媒体选题 | "美食文化" | 历史变迁 + 影响因素 + 选题角度 |
| 趋势推演 | "半导体产业链" | 事件因果链 + 关联分析 + 未来推演 |

## 五、三层认知模型

```
第一层：时间线（什么时候）
  └─ 按时间排列的事件流

第二层：因果网络（怎么关联的）  ← 核心竞争力
  └─ 事件之间的因果关系、竞争关系、
      包含关系、技术依赖关系

第三层：洞察行动（意味着什么）
  └─ AI 生成的趋势判断和行动建议
```

### 关联关系类型

| 关系 | 符号 | 示例 |
|---|---|---|
| 因果关系 | A → B | "Claude 4 发布" → "OpenAI 加速 GPT-5 开发" |
| 竞争关系 | A ↔ B | "GPT-5" ↔ "Claude 4" |
| 包含关系 | A ⊂ B | "端侧AI芯片" ⊂ "半导体产业链" |
| 技术依赖 | A ⇢ B | "AI应用" ⇢ "GPU供应" |
| 连锁反应 | A → B → C | "成本下降 → 创业潮 → 人才短缺" |

## 七、设计风格

Deta Surf 风格 + 阅读式信息流：

| 元素 | 规格 |
|---|---|
| 主色 | #009afc (Surf Blue) |
| 背景 | #f3f4f6 (Mist) |
| 卡片 | #ffffff (Paper) |
| 正文 | #000000 |
| 辅助 | #5b6882 |
| 章节标题 | Playfair Display 28px |
| 核心判断句 | Playfair Display 22px |
| 正文 | Inter / Noto Sans SC 17px / 行高 1.7 |
| 侧边注 | Inter 13px |
| 数据标注 | Inter uppercase 11px |

### UI 核心设计

1. **章节式叙事**：信息按逻辑分成 2-4 章，每章 2-4 个事件
2. **因果关系线**：事件卡片间用细线+箭头连接，标注关系类型
3. **侧边注释**：右侧 260px 放关联信息、数据来源，不打断阅读流
4. **大量留白**：章节之间 120px+ 间距，给人消化空间
5. **开头引导**：2-3 句 AI 摘要，建立认知框架
6. **结尾判断**：AI 提炼的洞察 + 分角色行动建议
