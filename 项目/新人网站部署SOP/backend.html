操作步骤

截图已整理为脱敏版操作示例，用于说明 D1 Database 创建和 Pages Functions Binding 配置流程。

1

进入 Cloudflare 的 D1 Database

从 Cloudflare 控制台左侧进入 D1 数据库区域，准备创建后端要使用的数据库。

2

点击 Create database

在 D1 Database 页面点击创建按钮，开始新建数据库。

3

回到 Workers & Pages 项目列表

找到当前网站对应的 Pages 项目，后续要把 D1 数据库绑定到这个项目。

4

填写 D1 数据库名称

数据库名称使用项目名或业务名，位置保持 Automatic，让 Cloudflare 自动选择。

5

确认数据库创建成功

创建完成后，D1 Database 列表里会出现新的数据库记录。

6

进入 Pages 项目的 Settings

进入网站项目后切换到 Settings，并找到 Functions 相关配置。

7

打开 Bindings 并添加绑定

在 Bindings 区域点击 Add，准备把 D1 数据库暴露给后端函数使用。

8

选择 D1 database 类型

绑定类型选择 D1 database，不要选 KV、R2 或其他存储类型。

9

选择刚创建的数据库并保存

变量名填写后端代码约定的名称，数据库选择刚创建的 D1 Database，然后保存配置。

上线检查

配置完成后，用下面几项快速确认后端页面已经切换到新的 Cloudflare D1 流程。

D1 Database 已创建

在 Cloudflare 的 D1 Database 列表里可以看到新数据库，名称和项目/业务保持一致。

Pages 项目已绑定数据库

在 Pages 项目的 Settings → Functions → Bindings 中，已添加 D1 database 绑定。

变量名与代码一致

Binding 的变量名必须和后端函数代码里读取的名称一致，否则运行时会找不到数据库。

保存后重新部署

绑定配置保存后，重新部署或触发一次 Pages 构建，让新配置进入生产环境。
