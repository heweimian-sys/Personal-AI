# 竞品调研网站 · 产品需求

> 本文档定义产品功能需求，是开发的依据。每次开发前请对照本文档。
> 本版为 v3 MVP 版本。

## 一、核心价值

打通"知道→做到"的信息差链路：

- 知道：自动收集、整合、提取现有信息
- 理解：将信息整理成时间线，发现趋势和关联
- 做到：输出行动方案

**核心痛点**：找不到信息、整合信息难、梳理信息难。

## 二、用户场景

### 场景 1：知识演变追踪
> "我想知道美食从古至今是怎么演变的"
- 输入"美食的演变"
- 系统搜索相关信息
- 生成时间线：古代烹饪方式 → 工业革命食品加工 → 现代外卖文化
- 建立关联：[[食品工业]]、[[饮食文化]]、[[健康影响]]
- 输出报告：演变脉络 + 对人的影响

### 场景 2：竞品调研
> "我想了解竞品 Notion 过去一年做了什么"
- 输入"Notion 产品动态"
- 系统搜索并整理
- 生成报告：产品更新时间线 + 功能变化 + 市场策略
- 建立关联：[[Notion]]、[[AI笔记工具]]、[[SaaS市场]]

### 场景 3：趋势梳理与推演
> "我想了解 AI 行业最近半年的趋势"
- 输入"AI 行业趋势"
- 生成时间线：GPT-5 发布、DeepSeek 开源、欧盟 AI 法案...
- 建立关联：[[GPT-5]]、[[DeepSeek]]、[[AI监管]]
- 输出报告：关键趋势 + 行动建议

### 场景 4：选题管理
> "我想做短视频赛道的选题"
- 输入"短视频赛道"
- 生成时间线：平台政策变化、热点事件、创作者动态
- 建立关联：[[抖音]]、[[快手]]、[[内容创作]]
- 输出报告：热点分析 + 选题建议

### 场景 5：趋势推演
> "我想看看新能源车市的趋势走向"
- 输入"新能源车市"
- 生成历史事件链
- 通过双向链接发现关联主题：[[比亚迪]]、[[充电桩]]、[[补贴政策]]
- 输出报告：历史脉络 + 关联分析 + 未来推演

## 三、MVP 功能范围

### 3.1 访问控制（简化版）

| 功能 | 说明 | 优先级 |
|---|---|---|
| 密码登录 | 输入访问密码获取令牌 | 必须 |
| 令牌校验 | 所有 API 需要有效令牌 | 必须 |

### 3.2 主题管理 (Topic)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 创建主题 | 输入关键词创建一个追踪主题 | 必须 |
| 查看主题列表 | 列出所有主题 | 必须 |
| 查看主题详情 | 显示主题的事件时间线 | 必须 |
| 删除主题 | 删除主题及其所有数据 | 必须 |
| 关联主题 | 创建/编辑主题时可关联其他主题 | 必须 |

### 3.3 双向链接 (Bidirectional Links) — 核心功能

| 功能 | 说明 | 优先级 |
|---|---|---|
| 手动链接 | 主题可以手动关联其他主题 | 必须 |
| 反向链接 | 被关联的主题自动显示来源链接 | 必须 |
| 链接展示 | 主题详情页显示正向和反向链接 | 必须 |
| 快速跳转 | 点击链接可跳转到关联主题 | 必须 |
| `[[主题名]]` 语法 | 事件内容中用此语法标记关联 | 必须 |
| AI 智能推荐 | AI 自动建议关联主题 | Backlog |
| 知识图谱可视化 | 网络图展示主题关联关系 | Backlog |

### 3.4 信息采集与研究 (Research)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 触发搜索 | 对主题执行一次搜索+AI分析 | 必须 |
| 时间范围筛选 | 支持不限/一天/一周/一月/一年 | 必须 |
| 自动去重 | 相同标题的事件不重复入库 | 必须 |
| 事件分类 | 自动标记类别：产品/政策/行业/融资/其他 | 必须 |
| 内容中支持链接 | 事件内容中可用 `[[主题名]]` 语法 | 必须 |

### 3.5 报告生成 (Report)

| 功能 | 说明 | 优先级 |
|---|---|---|
| 生成报告 | 基于主题事件，AI 生成总结报告 | 必须 |
| 查看报告列表 | 列出主题下的所有报告 | 必须 |
| 查看单个报告 | 显示报告全文 | 必须 |
| 报告内容 | 总览 + 关键趋势 + 行动建议 + Markdown 正文 | 必须 |
| 报告含关联 | 报告中展示关联主题的链接 | 必须 |

## 四、数据模型

### 主题 (Topic)
- keyword（搜索关键词）、display_name（显示名称）
- links（关联的主题列表）— 双向链接基础

### 主题链接 (TopicLink) — 双向链接关系表
- source_topic_id（来源主题）
- target_topic_id（被关联的主题）
- created_at（创建时间）

### 事件 (Event)
- occurred_at（发生日期）、title、summary、category、source
- content（事件详情，支持 `[[主题名]]` 语法）
- 归属某个主题

### 报告 (Report)
- report_type（报告类型）、content_md（Markdown正文）、structured（结构化数据）
- 归属某个主题

> MVP 不包含 Subscription、ChatSession、ChatMessage 模型，这些在 Backlog 中。

## 五、技术约束

| 项目 | 要求 |
|---|---|
| 搜索服务 | Firecrawl 自部署（完全免费，无限制） |
| AI 服务 | DeepSeek（国内直连，JSON Output） |
| 网络限制 | 国内可直连，不需要翻墙 |
| 响应速度 | 搜索+AI分析在 30 秒内完成 |
| 代码质量 | 核心链路有测试覆盖（搜索服务/AI服务/研究管道/链接服务） |
| 语言 | 所有界面、文档、AI 交互均使用中文 |

## 八、Backlog（验证通过后再启动）

以下功能本轮不做，先记录设计要点：

- **订阅式追踪**：APScheduler 定时任务 + Subscription 模型
- **对话式问答**：ChatSession + ChatMessage 模型，基于已积累的 events/reports 做 RAG 问答
- **完整多用户认证**：user_id + 注册登录
- **AI 智能推荐关联**：AI 自动建议关联主题
- **知识图谱可视化**：网络图展示主题关联关系
- **Docker Compose 全栈部署**：backend + frontend + db + firecrawl 一起打包
- **可视化看板**：Recharts 图表（事件密度趋势、分类占比等）
- **内容合规**：敏感词过滤、来源标注/版权、ICP 备案
